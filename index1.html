📒📒📒📒📒📒📒Code Analysis: A Node class is defined to represent each question and answer node. Each node has a VALUE (question or statement), an ANSWER (answer or feedback), and a list of CHILDREN, which contains the node's children, indicating possible follow-up questions or paths.

🐶🐶🐶🐶🐶🐶🐶Multiple instances of nodes are created using the Node class and these nodes are connected to each other through their children attribute to form a quiz tree. This tree is used to navigate different quiz paths based on the user's choices.

🐱🐱🐱🐱🐱🐱🐱Use the View and Button classes in discord.ui to create interactive interfaces. 🌛GuessOptionView and WrongView inherit from View and are used to display questions and options.🌞 GuessButton, YesButton, MaybeButton and WrongButton inherit from Button and are used to handle user click events. 🌛GuessOptionView displays the children of the current node as button options. ❌WrongView displays the options in case the user selects the wrong answer. ✅YesButton, MaybeButton and WrongButton are materialized as specific actions such as Confirm, Maybe and Error.

🔒🔒🔒🔒🔒🔒🔒FeedbackModal inherits from Modal and is used to collect user feedback under certain conditions. It contains a number of text input fields where the user can enter feedback.
